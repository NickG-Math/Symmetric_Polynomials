\subsubsection*{template$<$typename container\+\_\+t$>$\newline
class Symmetric\+\_\+\+Polynomials\+::\+Polynomial$<$ container\+\_\+t $>$}

Polynomials in multiple variables. 


\begin{DoxyTemplParams}{Template Parameters}

\begin{DoxyParagraph}{Requirements from container\+\_\+t}
The container must map pairs (degree,exponent) to scalars\+: $(d,e)\mapsto c$ represents the monomial $cx_1^{e_1}\cdots x_n^{e_n}$ of degree $d$. Specifically\+:
\begin{DoxyItemize}
\item {\ttfamily container\+\_\+t} must have a member {\ttfamily static} {\ttfamily constexpr} {\ttfamily bool} {\ttfamily ordered} that specifies whether it keeps the monomials in increasing order (yes if {\ttfamily ordered==1}, no otherwise)
\item If {\ttfamily ordered==1}, {\ttfamily container\+\_\+t} should be equivalent to {\ttfamily std\+::map$<$std\+::pair$<$typename exp\+\_\+t\+::deg\+\_\+t, exp\+\_\+t$>$}, scl\+\_\+t$>$
\item If {\ttfamily ordered==0}, {\ttfamily container\+\_\+t} should be equivalent to {\ttfamily std\+::unordered\+\_\+map$<$std\+::pair$<$typename exp\+\_\+t\+::deg\+\_\+t, exp\+\_\+t$>$}, scl\+\_\+t, hasher$>$
\item In particular {\ttfamily container\+\_\+t} must also have\+: {\ttfamily operator\mbox{[}$\,$\mbox{]}()}, {\ttfamily \mbox{\hyperlink{classSymmetric__Polynomials_1_1Polynomial_1_1const__iterator}{const\+\_\+iterator}}} with {\ttfamily \mbox{\hyperlink{classSymmetric__Polynomials_1_1Polynomial_af40d654efe6e0d1ac9904774a8a7994a}{begin()}}},\mbox{\hyperlink{classSymmetric__Polynomials_1_1Polynomial_aee8a3dc9d127069f4672416658b74442}{end()}} methods, and methods {\ttfamily \mbox{\hyperlink{classSymmetric__Polynomials_1_1Polynomial_a398889dc1151404048820a5c1b4baf5a}{insert()}}} and {\ttfamily erase()}.
\item Finally, it must have the typedefs {\ttfamily mapped\+\_\+type} (the scalar), {\ttfamily key\+\_\+type\+::first\+\_\+type} (the degree), {\ttfamily key\+\_\+type\+::second\+\_\+type} (the exponent). 
\end{DoxyItemize}
\end{DoxyParagraph}
\begin{DoxyParagraph}{Requirements from the exponent type {\ttfamily exp\+\_\+t} (i.e. {\ttfamily key\+\_\+type\+::second\+\_\+type})\+:}
The exponent must have functionality similar to {\ttfamily \mbox{\hyperlink{structSymmetric__Polynomials_1_1Standard__Variables}{Standard\+\_\+\+Variables}}} or {\ttfamily \mbox{\hyperlink{structSymmetric__Polynomials_1_1Half__Idempotent__Variables}{Half\+\_\+\+Idempotent\+\_\+\+Variables}}} Specifically\+:
\begin{DoxyItemize}
\item It must have a typedef {\ttfamily deg\+\_\+t} that represents the degree type (eg {\ttfamily int},uint64\+\_\+t)
\item It must have basic vector functionality (constructor that takes {\ttfamily int} {\ttfamily n} and produces exponent of 0\textquotesingle{}s with that number of variables {\ttfamily n}, operator \mbox{[}\mbox{]}...)
\item It should have an operator + to be used in the product of monomials and an operator -\/ to be used in the division of monomials (if products/divisions need to be used).
\item If {\ttfamily ordered==1} it needs to have an operator $<$ (implementing a lexicographic ordering) and otherwise it needs to have an operator () implementing a hash function
\item It may also optionally have a {\ttfamily deg\+\_\+t} {\ttfamily degree()} function that compute the degree of the exponent
\item It may also optionally have a {\ttfamily std\+::string} {\ttfamily static} {\ttfamily name(int,int)} function that prints the names of the variables (first parameter is the index of the variable, second is the total number of variables).
\item If {\ttfamily exp\+\_\+t} does not have a {\ttfamily degree()} function then the user needs to provide the dimensions of the variables through a {\ttfamily deg\+\_\+t$\ast$} 
\item If {\ttfamily exp\+\_\+t} does not have a {\ttfamily static} {\ttfamily name(int,int)} function then the user needs to provide the names of the variables through a {\ttfamily std\+::string$\ast$} 
\end{DoxyItemize}
\end{DoxyParagraph}
