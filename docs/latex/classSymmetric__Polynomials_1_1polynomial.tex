\subsubsection*{template$<$typename container\+\_\+t$>$\newline
class Symmetric\+\_\+\+Polynomials\+::\+Polynomial$<$ container\+\_\+t $>$}

Polynomials in multiple variables. 


\begin{DoxyTemplParams}{Template Parameters}
{\em container\+\_\+t} & Specifies the container for the monomials in the polynomial eg \mbox{\hyperlink{structSymmetric__Polynomials_1_1default__container}{default\+\_\+container}}$<$int, Standard\+\_\+\+Variables$<$$>$, 1$>$ or \mbox{\hyperlink{structSymmetric__Polynomials_1_1default__container}{default\+\_\+container}}$<$float, Half\+\_\+\+Idempotent\+\_\+\+Variables$<$$>$, 0$>$\\
\hline
\end{DoxyTemplParams}
In essence the container must map pairs (degree,exponent)= $(d,e)$ to scalars= $c$ representing the monomial $cx_1^{e_1}\cdots x_n^{e_n}$ of degree $d$.

Requirements from container\+\_\+t\+:


\begin{DoxyItemize}
\item container\+\_\+t must have a static constexpr bool variable ordered that specifies whether it keeps the monomials in increasing order (yes if ordered=1, no if ordered=0)
\item If ordered=1, container\+\_\+t should be equivalent to std\+::map$<$std\+::pair$<$typename exp\+\_\+t\+::deg\+\_\+t, exp\+\_\+t$>$, scl\+\_\+t$>$
\item If ordered=0, container\+\_\+t should be equivalent to std\+::unordered\+\_\+map$<$std\+::pair$<$typename exp\+\_\+t\+::deg\+\_\+t, exp\+\_\+t$>$, scl\+\_\+t, hasher$>$
\end{DoxyItemize}

In particular container\+\_\+t must also have\+: operator\mbox{[}$\,$\mbox{]}(), \mbox{\hyperlink{classSymmetric__Polynomials_1_1Polynomial_1_1const__iterator}{const\+\_\+iterator}} with \mbox{\hyperlink{classSymmetric__Polynomials_1_1Polynomial_af40d654efe6e0d1ac9904774a8a7994a}{begin()}},\mbox{\hyperlink{classSymmetric__Polynomials_1_1Polynomial_aee8a3dc9d127069f4672416658b74442}{end()}} methods, and methods \mbox{\hyperlink{classSymmetric__Polynomials_1_1Polynomial_a398889dc1151404048820a5c1b4baf5a}{insert()}} and erase().

Finally, it must have the typedefs mapped\+\_\+type (the scalar), key\+\_\+type\+::first\+\_\+type (the degree), key\+\_\+type\+::second\+\_\+type (the exponent).

The exponent type key\+\_\+type\+::second\+\_\+type, henceforth exp\+\_\+t, must have functionality similar to \mbox{\hyperlink{structSymmetric__Polynomials_1_1Standard__Variables}{Standard\+\_\+\+Variables}} or \mbox{\hyperlink{structSymmetric__Polynomials_1_1Half__Idempotent__Variables}{Half\+\_\+\+Idempotent\+\_\+\+Variables}}

Requirements of exp\+\_\+t (i.\+e. key\+\_\+type\+::second\+\_\+type)\+:


\begin{DoxyItemize}
\item A typedef deg\+\_\+t that represents the degree type (eg int or uint64\+\_\+t)
\item Basic vector functionality (constructor that takes int n and produces exponent of 0\textquotesingle{}s with that number of variables n, operator\mbox{[}\mbox{]})
\item It should have an operator + to be used in the product of monomials and an operator -\/ to be used in the division of monomials (if products/divisions need to be used).
\item If ordered=1 it needs to have an operator $<$ (implementing a lexicographic ordering) and if ordered=0 it needs to have an operator () implementing a hash function
\item It may also optionally have a degree() function and a static name(int,int) function that compute the degree of the exponent and print the names of the variables (first parameter is the index of the variable, second is the total number of variables).
\item if exp\+\_\+t does not have a degree() function then the user needs to provide the dimensions of the variables through a deg\+\_\+t$\ast$ (the user also needs to make sure that this pointer is not invalidated as long as the polynomial is used)
\item if exp\+\_\+t does not have a static name(int,int) function then the user needs to provide the names of the variables through an std\+::string$\ast$ (the user also user needs to make sure that this pointer is not invalidated as long as the polynomial is used) 
\end{DoxyItemize}
