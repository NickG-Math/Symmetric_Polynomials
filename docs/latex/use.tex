Everything in this library is under the namespace {\ttfamily \mbox{\hyperlink{namespaceSymmetric__Polynomials}{Symmetric\+\_\+\+Polynomials}}}. For all the following code examples to work, start with \begin{DoxyVerb}using namespace Symmetric_Polynomials;

\hypertarget{use_mono}{}\doxysubsubsection{Polynomials}\label{use_mono}
A polynomial in multiple variables can be default constructed as\+: \begin{DoxyVerb}Polynomial<default_container<int, Standard_Variables<>>> p;
\end{DoxyVerb}


Let us explain the template parameters\+:
\begin{DoxyItemize}
\item {\ttfamily int} is the type of the scalar coefficients, so it can be replaced by {\ttfamily float} etc.
\item {\ttfamily Standard\+\_\+\+Variables$<$$>$} specifies that our variables are $x_1,...,x_n$ with degrees $|x_i|=1$ and no relations. It can be replaced by {\ttfamily Half\+\_\+\+Idempotent\+\_\+\+Variables} that specify variables $x_1,...,x_n,y_1,...,y_n$ with degrees $|x_i|=1, |y_i|=0$ and relations $y_i^2=y_i$.
\item The {\ttfamily default\+\_\+container} specifies the container used to store the monomials in the polynomial. See the advanced subsection below.
\end{DoxyItemize}

To initialize a polynomial, provide the desired number of variables $n$. For example, if $n=3$\+: \begin{DoxyVerb}p=Polynomial<default_container<int, Standard_Variables<>>>(3);
\end{DoxyVerb}


Next, insert monomials $cx_1^{a_1}\cdots x_n^{a_n}$ by providing the exponent vector $[a_1,...,a_n]$ and the coefficient $c$. Eg\+: \begin{DoxyVerb}p.insert({0,1,4},7);
\end{DoxyVerb}


inserts the monomial $7x_2x_3^4$ in {\ttfamily p}. If we further insert \begin{DoxyVerb}p.insert({1,1,2},-8);
\end{DoxyVerb}


then {\ttfamily p} becomes $-8x_1x_2x_3^2+7x_2x_3^4$ and printing it to console verifies that\+: \begin{DoxyVerb}std::cout << p << "\n";
\end{DoxyVerb}


It is the user\textquotesingle{}s responsibility to make sure that
\begin{DoxyItemize}
\item the number of variables is the length of the exponent vector being inserted
\item the coefficient provided is never $0$
\item not attempt to insert a monomial with an already existing exponent vector (doing so does not change the polynomial)
\end{DoxyItemize}

Polynomials can be added, subtracted and multiplied by binary operators {\ttfamily +,-\/,$\ast$} eg\+: \begin{DoxyVerb}std::cout << (p+(p^2))<< "\n";
\end{DoxyVerb}


will print

The class {\ttfamily Symmetric\+\_\+\+Basis\+\_\+\+Default} allows us to transform polynomials on $x_i$ variables ({\ttfamily Standard\+\_\+\+Variables}) into polynomials on the elementary symmetric polynomials $e_i=\sigma_i$ ({\ttfamily Elementary\+\_\+\+Symmetric\+\_\+\+Variables}). For example\+: \begin{DoxyVerb}Symmetric_Basis_Default<int> SB(2);

Polynomial<default_container<int, Standard_Variables<>>> poly(2);

poly.insert({1,2},2);

poly.insert({2,1},2);

Polynomial<default_container<int, Elementary_Symmetric_Variables<>>>  new_poly= SB(poly);

Polynomial<default_container<int, Standard_Variables<>>>  new_new_poly= SB(new_poly);

std::cout << poly << "\n";

std::cout<< new_poly << "\n";

std::cout << new_new_poly << "\n";
\end{DoxyVerb}


does the following\+: First it sets {\ttfamily poly} to $2x_1x_2^2+2x_1^2x_2$, then transforms it in $e_i$ variables as {\ttfamily new\+\_\+poly} which is $2e_1e_2$ and then transforms that back to $x_i$ variables as {\ttfamily new\+\_\+new\+\_\+poly} which is $2x_1x_2^2+2x_1^2x_2$ (i.\+e. the original polynomial) and finally prints the three polynomials. This all means that \$\$2x\+\_\+1x\+\_\+2$^\wedge$2+2x\+\_\+1$^\wedge$2x\+\_\+2=2e\+\_\+1e\+\_\+2\$\$\hypertarget{use_halfidem}{}\doxysubsubsection{Half idempotent relations}\label{use_halfidem}
When we have the \char`\"{}half idempotent relation\char`\"{} i.\+e. variables $x_i,y_i$ with $y_i^2=y_i$ (as specified by {\ttfamily Half\+\_\+\+Idempotent\+\_\+\+Variables}), the generators are $\alpha_i, c_i, \gamma_{s,t}$ (see \mbox{\hyperlink{math}{The Math}}) or {\ttfamily Idem\mbox{[}i\mbox{]}}, {\ttfamily Chern\mbox{[}i\mbox{]}} and {\ttfamily Twisted\+Chern\mbox{[}s\mbox{]}\mbox{[}t\mbox{]}} respectively as they are used in the program. We order them as $\alpha_1,...,\alpha_n,c_1,...,c_n, \gamma_{1,1},...,\gamma_{1,n-1},\gamma_{2,1},...,\gamma_{n-1,1}$.

The class {\ttfamily Half\+\_\+\+Idempotent\+\_\+\+Basis\+\_\+\+Default} allows us to transform polynomials on $x_i,y_i$ variables ({\ttfamily Half\+\_\+\+Idempotent\+\_\+\+Variables}) into polynomials on the $\alpha_i,c_i,\gamma_{s,i}$ ({\ttfamily Twisted\+\_\+\+Chern\+\_\+\+Variables}).

The code \begin{DoxyVerb}Half_Idempotent_Basis_Default<int> HB(2);

Polynomial<default_container<int, Half_Idempotent_Variables<>>> poly2(4);

poly2.insert({ 1,0,1,0 }, 2);

poly2.insert({ 0,1,0,1 }, 2);

auto new_poly2= HB(poly2);

auto new_new_poly2= HB(new_poly2);

std::cout << poly2 << "\n";

std::cout << new_poly2 << "\n";

std::cout << new_new_poly2 << "\n";
\end{DoxyVerb}


sets {\ttfamily poly} to be $x_1y_1+x_2y_2$, transforms it into {\ttfamily new\+\_\+poly} which is $2\alpha_1 c_1 + -2\gamma_{1,1}$ and then transforms it to {\ttfamily new\+\_\+new\+\_\+poly} in the original variables, which is equal to {\ttfamily poly}, before printing their names. Thus\+:

\[x_1y_1+x_2y_2=-2\gamma_{1,1}+2\alpha_1 c_1\]

To print the relations amongst $\alpha_i,c_i,\gamma_{s,j}$ simply use\+: \begin{DoxyVerb}print_half_idempotent_relations<>(3,1,1);
\end{DoxyVerb}


The $3$ here corresponds to the half the number of variables\+: $x_1,x_2,x_3,y_1,y_2,y_3$ and can be replaced by any positive integer.

Finally, \mbox{\hyperlink{Demo_8cpp}{Demo.\+cpp}} contains another function, {\ttfamily write\+\_\+pontryagin\+\_\+\+C2\+\_\+in\+\_\+terms\+\_\+of\+\_\+\+Chern\+\_\+classes}. This prints the expressions of the twisted Pontryagin/symplectic classes given by

\[\pi_{s,t}=\kappa_{s,t}=\sum_{1\le i_1< \cdots< i_s\le n\\ 1\le j_1< \cdots< j_t\le n\\ i_u\neq j_v}x^2_{i_1}....x^2_{i_s}y_{j_1}\cdots y_{j_t}\]

in terms of the $\alpha_i,c_i,\gamma_{s,t}$. For example, try running \begin{DoxyVerb}write_pontryagin_C2_in_terms_of_Chern_classes(5);
\end{DoxyVerb}
\hypertarget{use_adv}{}\doxysubsubsection{Advanced}\label{use_adv}
The {\ttfamily default\+\_\+container$<$scl\+\_\+t, exp\+\_\+t, ordered$>$} specifies the data structure that should be used to internally store the monomials of the polynomial. This must have the features of an ordered/unordered map (degree,exponent)-\/$>$coefficient. The boolean template parameter {\ttfamily ordered} specifies whether we want to use {\ttfamily std\+::map} (default) or {\ttfamily std\+::unordered\+\_\+map}. When unordered maps are used, it\textquotesingle{}s important to\+:
\begin{DoxyItemize}
\item Reserve space for the amount of monomials to be inserted by eg\+: {\ttfamily p.\+reserve(5);} if we expect to store $5$ monomials
\item Keep in mind that when iterating through the polynomial, the monomials won\textquotesingle{}t be ordered. Same applies when printing the polynomial. In general, using unordered maps is faster especially with a good hashing function (see \mbox{\hyperlink{General_8hpp}{General.\+hpp}} for two possible hashing functions that can be used).
\end{DoxyItemize}

You can also use your own container instead of the {\ttfamily default\+\_\+container}. See the file \mbox{\hyperlink{Polynomials_8hpp}{Polynomials.\+hpp}} for the exact requirements from a custom container.

Finally, the {\ttfamily Symmetric\+\_\+\+Basis\+\_\+\+Default} and {\ttfamily Half\+\_\+\+Idempotent\+\_\+\+Basis\+\_\+\+Default} are aliases for {\ttfamily Symmetric\+\_\+\+Basis} and {\ttfamily Half\+\_\+\+Idempotent\+\_\+\+Basis} that accept containers as template parameters for extra customization.
